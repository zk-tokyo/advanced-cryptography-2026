# Week 0 サンプル課題

Week 0には、自動採点の動作確認用に2種類のサンプル課題があります。
どちらか1つを選んで提出してください。

- 文章課題: LLMが答案を読んで採点します。
- Python穴埋め課題: CIが提出されたPythonコードを実行して採点します。

## 課題A: コミットメントの文章課題

次の3問のうち、いずれか1問を選んで回答してください。
答案の冒頭に、選んだ問題番号を `選択: 問題1`、`選択: 問題2`、または
`選択: 問題3` のように明記してください。

### 問題1: 難易度 低

コミットメント方式におけるbinding性とhiding性について説明してください。

答案では、次の点に触れてください。

1. binding性が何を保証する性質かを述べる。
2. hiding性が何を保証する性質かを述べる。
3. それぞれの性質が、コミットメントのどの段階で重要になるかを説明する。

### 問題2: 難易度 中

次のトイコミットメント構成が、ハッシュ関数 `H` の衝突困難性に依存してbindingである
理由を説明してください。

```text
Com(m; r) = H(m || r)
```

答案では、次の点に触れてください。

1. この構成におけるbinding性の意味を述べる。
2. 2つの異なる有効なopeningが存在すると、`H` の衝突が得られることを説明する。
3. このトイ構成の限界を少なくとも1つ述べる。

### 問題3: 難易度 高

素数位数群 `G` の生成元 `g, h` を用いたPedersenコミットメント
`Com(m; r) = g^m h^r` について説明してください。ただし、`log_g h` はコミッターに
知られていないものとします。

答案では、次の点に触れてください。

1. この構成がperfect hidingである理由を説明する。
2. `log_g h` を知っているとbinding性を破れる理由を説明する。
3. 離散対数問題の困難性とcomputational bindingの関係を説明する。

### 提出方法

答案を `answer.md` として提出してください。複数の問題に回答した場合でも、
採点対象は1問分だけです。

## 課題B: Python穴埋め課題

`week0/exercises/templates/solution.py` を参考にして、次の2つの関数を実装してください。

```python
def mod_pow(base: int, exponent: int, modulus: int) -> int:
    ...


def mod_inverse(a: int, modulus: int) -> int:
    ...
```

### 仕様

- `mod_pow(base, exponent, modulus)` は `base ** exponent mod modulus` を返す。
- `mod_inverse(a, modulus)` は `(a * x) % modulus == 1` を満たす `x` を返す。
- 逆元が存在しない場合、`mod_inverse` は `ValueError` を投げる。
- Python標準ライブラリだけを使う。

### 提出方法

実装したファイルを `solution.py` として提出してください。
